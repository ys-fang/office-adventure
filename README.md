# 均一辦公室探險

午休時間，你決定好好探索辦公室的每個角落。

每位同事建立一個辦公室場景 -- 茶水間、會議室、頂樓天台、被遺忘的儲藏室。所有場景會自動串聯成一個可探索的世界。每個人的場景就是這張冒險地圖的一塊拼圖。

**這是「第一次 PR 工作坊」專案** -- 讓非技術背景的夥伴也能體驗 GitHub 協作流程。

---

## 玩法

- **探索辦公室**：在地圖上往東西南北移動，走進不同的場景
- **做出選擇**：每個場景裡有 3-4 個互動選項，但你每次只能選一個
- **集字過關**：辦公室裡藏著「開」「源」「人」三個字，找齊就能獲得「開源人」匾額
- **性格側寫**：你的選擇會累積成四種特質（行動派 / 觀察者 / 冒險家 / 暖心人），最後揭曉你的探險者類型
- **你的故事**：所有選擇串成一篇專屬你的辦公室探險故事，可以複製分享

## 快速開始：建立你的場景

### 第 1 步：Fork 這個專案

1. 點擊頁面右上角的 **Fork** 按鈕
2. 直接點 **Create fork**（不需要改設定）
3. 等幾秒，你會進入自己的副本

### 第 2 步：複製模板

1. 進入你 Fork 後的專案
2. 點進 `contributors/` 資料夾
3. 點開 **`_template.md`** 這個檔案
4. 點右上角的 **Raw** 按鈕，全選複製（Ctrl+A / Cmd+A → Ctrl+C / Cmd+C）

### 第 3 步：建立你的場景檔案

1. 回到 `contributors/` 資料夾
2. 點右上方 **Add file** > **Create new file**
3. 檔案名稱用英文或拼音，例如 `tea-room.md`、`rooftop.md`（不要有空格）
4. 在編輯區貼上剛剛複製的模板內容
5. 把模板裡的文字改成你的創作

### 第 4 步：寫你的場景

模板長這樣，改成你自己的內容：

```
# 你的場景名稱

描述走進這個場景時看到的樣子。

## 可以做的事

做某件事：描述做了之後的結果。
檢查某個東西：描述你發現了什麼。*紀念品名稱*
嘗試某個行動：描述發生了什麼。
```

#### 寫作建議

1. 第一行用 `#` 加上場景名稱
2. 場景描述寫在標題下面
3. 用 `## 可以做的事` 開始寫互動選項（3 到 4 個）
4. 每個選項格式：`動作：結果`
5. 想加收藏品就在結尾加 `*名稱*`（選加，不是必要的）

### 第 5 步：提交

1. 頁面下方 **Commit changes** 區塊
2. commit message 填：`add 場景名稱`
3. 選 **Commit directly to the main branch**
4. 點 **Commit changes**

### 第 6 步：發起 Pull Request

1. 回到你 Fork 的專案首頁
2. 看到 **This branch is 1 commit ahead** 的提示
3. 點 **Contribute** > **Open pull request**
4. 點 **Create pull request**
5. 等維護者合併，你的場景就會出現在辦公室探險世界裡了！

## 給維護者的說明

合併 PR 就好！`contributors/index.md` 會由 GitHub Actions 自動更新，不需要手動維護。

---

## 部署

1. 在 GitHub repo 的 Settings > Pages
2. Source 選 **Deploy from a branch**
3. Branch 選 `main`，資料夾選 `/ (root)`
4. 儲存後等 1-2 分鐘，網站就上線了

管理者模式：在網址後加 `?admin` 可以查看完整的場景地圖配置。

---

## 技術細節

- 純 HTML/CSS/JS，無需建置步驟，一個 `index.html` 搞定
- 透過 GitHub Pages 部署
- 每個參與者一個獨立 `.md` 檔，不會產生合併衝突
- 房間位置由檔名 hash 決定，新增房間不影響現有配置
- 方向性導航：往北就是往北，回頭一定回到原地
- 打錯格式不會壞 -- 系統會盡力解析，解析不了的部分會跳過
